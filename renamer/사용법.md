# Renamer 사용법

파일명 일괄 변경 도구입니다. 인터넷 연결 없이 100% 내 노트북에서만 동작하며, 파일이 외부로 나가지 않습니다. **macOS·Windows 모두 지원**합니다.

---

## 1. 설치 (비개발자용) — macOS

1. 전달받은 **`Renamer_0.1.0_aarch64.dmg`** 를 더블클릭합니다.
2. 창이 열리면 **Renamer 아이콘을 옆의 `Applications` 폴더로 끌어다 놓습니다.**
   ⚠️ **DMG 창 안에서 바로 더블클릭하면 "손상되었습니다" 오류가 납니다.** 꼭 응용 프로그램 폴더로 **복사한 뒤** 실행하세요.
3. 이제 Launchpad나 응용 프로그램 폴더에서 실행할 수 있습니다.

> **아이콘이 Apple 실리콘(M1~) 맥용입니다.** 인텔 맥이라면 별도 빌드가 필요합니다.

### ⚠️ "'Renamer'은(는) 손상되었기 때문에 열 수 없습니다" 가 뜨면

앱이 실제로 손상된 게 **아닙니다.** 애플 서명이 없는 앱을 인터넷에서 받으면 macOS(특히 M1~ 맥)가 이렇게 막습니다. 격리 표시만 떼면 됩니다.

**방법 (가장 확실) — 터미널에서 한 줄 붙여넣기:**

1. `응용 프로그램 → 유틸리티 → 터미널` 을 엽니다.
2. 아래를 복사해 붙여넣고 Enter:
   ```
   xattr -cr /Applications/Renamer.app
   ```
3. 이제 **Renamer** 를 더블클릭하면 열립니다. (한 번만 하면 됩니다.)

> 이 앱은 파일을 외부로 보내지 않는 로컬 도구입니다. "손상됨" 은 서명이 없어서 뜨는 것일 뿐입니다.

---

## 1-1. 설치 (비개발자용) — Windows

1. 전달받은 **`Renamer_0.1.0_x64-setup.exe`**(또는 `.msi`)를 더블클릭합니다.
2. 안내에 따라 설치하면 시작 메뉴에 **Renamer** 가 생깁니다.
3. Windows 10/11에는 필요한 WebView2 런타임이 기본 내장되어 있어 추가 설치가 없습니다.

### ⚠️ 처음 열 때 "Windows의 PC 보호" 파란 창이 뜨면

애플과 마찬가지로 서명이 없어 뜨는 경고입니다.

1. 파란 창에서 **[추가 정보]** 를 클릭합니다.
2. 아래에 나타나는 **[실행]** 버튼을 누릅니다.

---

## 2. 기본 사용 흐름

1. **파일 추가** — 파일이나 폴더를 창의 점선 영역에 **드래그앤드롭** 하거나, `파일 선택…` / `폴더 선택…` 버튼을 누릅니다.
   - 폴더를 넣으면 안의 파일이 모두 추가됩니다.
   - `하위 폴더 포함` 을 켜면 폴더 안의 폴더까지 훑습니다.
   - `.DS_Store` 같은 숨김 파일은 자동으로 빠집니다.
2. **모드 선택** — `날짜 포맷 변환` 또는 `순번 붙이기`.
3. **미리보기 확인** — 아래 목록에서 `변경 전 → 변경 후` 를 눈으로 확인합니다.
4. **적용하기** — 버튼을 누르면 실제로 이름이 바뀝니다.
5. 잘못했으면 바로 **되돌리기** (마지막 1회 복원).

목록의 각 줄 왼쪽 **×** 를 누르면 그 파일만 대상에서 뺄 수 있습니다.

---

## 3. 모드 1 — 날짜 포맷 변환

파일명 맨 앞의 `YYMMDD_` 를 `YYYYMMDD_` 로 바꿉니다.

| 변경 전 | 변경 후 |
|---|---|
| `260520_삼청동.jpg` | `20260520_삼청동.jpg` |
| `260701_강남.png` | `20260701_강남.png` |

- 세기 접두어는 기본 `20`(2000년대)이며, 필요하면 `19`로 바꿀 수 있습니다.
- 존재하지 않는 날짜(예: `260231_…`)는 **변환 불가**로 표시하고 건너뜁니다.
- 이미 `YYYYMMDD_` 형식인 파일은 **이미 변환됨**으로 자동 제외됩니다(중복 변환 방지).
- 확장자는 항상 유지됩니다.

---

## 4. 모드 2 — 순번 붙이기

파일을 정렬한 뒤 `01`, `02` … 순번과 고정 문자를 붙입니다.

**예) 고정 문자 `_강남촬영`, 위치 맨 앞:**

| 변경 전 | 변경 후 |
|---|---|
| `IMG_2031.jpg` | `01_IMG_2031_강남촬영.jpg` |
| `IMG_2029.jpg` | `02_IMG_2029_강남촬영.jpg` |

- **정렬 기준**: 생성일 / 수정일 / 파일명
- **정렬 방향**: 최신순(최신=01) / 오래된순(오래된=01)
- **순번 위치**: 순번을 맨 앞 또는 맨 뒤(확장자 앞)에 배치
- **시작 번호**: 기본 1
- **자릿수**: `자동`(파일 100개 이상이면 `001`) 또는 `수동`
- **구분자**: 기본 `_`
- **완전 대체**: 켜면 원본 이름 대신 `순번+고정 문자`만 사용

> 순번은 위치 토글대로 맨 앞/맨 뒤에 붙고, **고정 문자는 원본 이름 뒤(확장자 앞)에** 붙습니다.
> PRD의 예시(`01_IMG_2031_강남촬영.jpg`)를 그대로 재현한 규칙입니다.

---

## 5. 안전장치

- **미리보기 필수**: 적용 전 결과를 모두 볼 수 있습니다.
- **충돌 방지**: 바뀐 이름끼리 겹치면(같은 폴더) 적용 버튼이 잠기고 빨간색으로 표시됩니다.
- **기존 파일 보호**: 대상 폴더에 같은 이름이 이미 있으면, 하단의 `이름 충돌 시` 설정에 따라
  **건너뛰기** 하거나 `(1)` **접미어**를 붙입니다.
- **되돌리기**: 마지막 적용 1회를 원래 이름으로 복원합니다.
- 일부만 실패해도 성공분은 유지되며, 실패 목록과 사유를 알려줍니다.

## 6. 개발자용 — 빌드 방법

필요 도구: Node.js, Rust(cargo).

```bash
cd renamer
npm install
npm run tauri dev     # 개발 중 실행
npm run tauri build   # 배포용 .app / .dmg 생성
```

생성물 위치:
- 앱: `src-tauri/target/release/bundle/macos/Renamer.app`
- 설치본: `src-tauri/target/release/bundle/dmg/Renamer_0.1.0_aarch64.dmg`

테스트:
```bash
node --test tests/logic.test.mjs      # 리네이밍 규칙(JS)
cd src-tauri && cargo test            # 파일 처리(Rust)
```

### Windows 설치본 만들기 — GitHub Actions (권장)

이 Mac에서는 Windows 실행 파일을 만들 수 없어(WebView2·MSVC·설치 관리자 필요), 클라우드 빌드를 씁니다.
저장소 루트의 `.github/workflows/build.yml` 이 이 일을 자동으로 합니다.

1. (최초 1회) 저장소를 GitHub에 올립니다. 프로젝트 루트에서:
   ```bash
   git init && git add . && git commit -m "Renamer 초기 버전"
   gh repo create renamer --private --source=. --push   # gh CLI 사용 시
   ```
   (`gh auth login` 이 안 되어 있으면 먼저 로그인. GitHub 웹에서 새 저장소를 만들어 push 해도 됩니다.)
2. 푸시하면 **Actions** 탭에서 빌드가 돌아갑니다. (수동 실행: Actions → *Build installers* → *Run workflow*)
3. 빌드가 끝나면 실행 결과 화면 하단 **Artifacts** 에서 내려받습니다.
   - `renamer-windows` → `.msi` / `-setup.exe`
   - `renamer-macos` → `.dmg`

### 버전 릴리스 — 태그를 밀면 Release 페이지에 자동 첨부

배포용 버전을 낼 땐 태그를 밀면 됩니다. 빌드 후 설치본이 **Releases** 페이지에 자동으로 올라갑니다.
```bash
git tag v1.0.0
git push origin v1.0.0
```
→ 저장소의 **Releases** 탭에 `v1.0.0` 릴리스가 생기고 `.exe`·`.msi`·`.dmg` 가 첨부됩니다.

### ⚠️ 비개발자 팀원에게 나눠줄 때 (중요)

- 저장소가 **비공개(private)** 면, Release/Artifacts 링크는 **GitHub 계정과 저장소 접근 권한이 있어야** 열립니다.
  비개발자 팀원에게는 부담이 됩니다.
- **권장 방식**: 저장소는 비공개로 두고, **본인이** Releases 페이지에서 `.exe`(윈도우)·`.dmg`(맥)를 내려받은 뒤
  그 **설치 파일을 팀원에게 직접 전달**하세요 (Slack·카톡·구글 드라이브·에어드롭 등). 팀원은 GitHub를 몰라도 됩니다.
- 만약 저장소를 **공개(public)** 로 두면, Release 링크만으로 누구나 로그인 없이 바로 받을 수 있습니다.
  (단, 소스 코드도 공개됩니다.)

### Windows PC에서 직접 빌드하려면

Node.js, Rust, **Microsoft C++ Build Tools**(Desktop development with C++), WebView2(윈도우 기본 내장)를 설치한 뒤:
```bash
cd renamer
npm install
npm run tauri build
```
결과물: `src-tauri\target\release\bundle\nsis\*.exe`, `...\msi\*.msi`
