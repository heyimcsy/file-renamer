# PRD — 맥용 파일명 일괄 변경 도구 (가칭: Renamer)

- 문서 버전: v0.1 (초안)
- 작성일: 2026-07-11
- 대상 플랫폼: macOS (개인 노트북 로컬 실행)
- 배포 방식: 팀원끼리 파일 하나(또는 앱 하나)를 공유해서 각자 로컬에서 실행

---

## 1. 배경 및 문제 정의

날짜 기반 파일명(`260520_삼청동`, `260701_강남`)으로 파일을 관리하고 있으나, 다음과 같은 반복 작업이 수작업으로 이루어지고 있다.

1. **날짜 포맷 통일**: `YYMMDD_이름` 형식을 `YYYYMMDD_이름`으로 일괄 변경해야 하는 경우가 잦다. (예: `260520_삼청동` → `20260520_삼청동`)
2. **순번 붙이기**: 파일 생성일 순서대로 `01_`, `02_` … 같은 순번과 고정 문자를 파일명 앞 또는 뒤에 붙여야 하는 경우가 있다.

Finder에서 하나씩 이름을 바꾸는 방식은 파일이 수십~수백 개일 때 비효율적이고 실수(중복, 누락, 오타)가 발생하기 쉽다. 개발 지식이 없는 팀원도 쓸 수 있는, 설치가 간단한 로컬 도구가 필요하다.

## 2. 목표 / 비목표

### 목표
- 드래그앤드롭 또는 폴더 선택만으로 파일명을 일괄 변경할 수 있다.
- 변경 전 **미리보기**를 제공해 실수를 방지한다.
- 실행 파일(또는 스크립트) 하나를 공유하면 각자 노트북에서 바로 쓸 수 있다.
- 인터넷 연결 없이 100% 로컬에서 동작한다. (파일이 외부로 나가지 않음)

### 비목표 (v1에서 제외)
- Windows 지원 (macOS 우선, 추후 검토)
- 클라우드 저장소(구글 드라이브 등) 내 파일 직접 변경
- 파일 내용 편집, 형식 변환
- 복잡한 정규식 기반 커스텀 규칙 빌더 (v2 후보)

## 3. 사용자 및 사용 시나리오

**대상 사용자**: 촬영본/작업 파일을 날짜 기반으로 관리하는 팀원. 터미널 사용 경험이 없어도 사용 가능해야 한다.

**시나리오 A — 날짜 포맷 통일**
1. 앱을 실행하고 파일 20개를 창에 드래그앤드롭한다.
2. "날짜 포맷 변환 (YYMMDD → YYYYMMDD)" 모드를 선택한다.
3. 미리보기에서 `260520_삼청동.jpg → 20260520_삼청동.jpg` 를 확인한다.
4. [적용] 버튼을 누르면 즉시 변경된다.

**시나리오 B — 폴더 통째로 순번 붙이기**
1. 폴더 하나를 창에 드롭한다. (하위 폴더 포함 여부 선택 가능)
2. "순번 붙이기" 모드를 선택하고 아래를 설정한다.
   - 정렬 기준: 생성일 최신순
   - 고정 문자: `_강남촬영`
   - 순번 위치: 맨 앞
3. 미리보기에서 `01_IMG_2031_강남촬영.jpg`, `02_IMG_2029_강남촬영.jpg` … 를 확인하고 적용한다.

### 4.1 파일 입력
| ID | 요구사항 | 우선순위 |
|---|---|---|
| F-01 | 파일 여러 개를 창에 드래그앤드롭으로 추가할 수 있다 | P0 |
| F-02 | 폴더를 드롭하면 폴더 내 파일 전체가 추가된다 | P0 |

| F-12 | 이미 `YYYYMMDD_` 형식인 파일은 변경 대상에서 자동 제외한다 (중복 변환 방지) | P0 |
| F-13 | 확장자는 항상 유지한다 | P0 |

| ID | 요구사항 | 우선순위 |

## 6. 기술 방향 (제안)

핵심 제약: **파일 생성일(birthtime) 읽기**와 **실제 파일명 변경**이 필요하다. 브라우저 단독(단일 HTML 파일)로는 생성일을 읽을 수 없어 제외한다.

| 옵션 | 장점 | 단점 | 배포 방식 |
|---|---|---|---|
| **A. Tauri (권장)** | 네이티브 .app, 가볍고(수 MB) 빠름, 드래그앤드롭·생성일 모두 지원 | Rust 빌드 환경 필요 (개발자만) | .dmg / .app 압축 공유 |
| B. Electron | 생태계 성숙, 개발 빠름 | 앱 용량 큼(100MB+) | .dmg 공유 |
| C. Python + Tkinter | 코드 한 파일로 공유 가능 | 드래그앤드롭에 외부 패키지 필요, 실행 진입장벽(터미널) | .py 또는 py2app 패키징 |

**권장안: A (Tauri)**
- 팀원은 .app을 받아 응용 프로그램 폴더에 넣기만 하면 됨 (비개발자 친화적)
- 서명 없는 앱의 Gatekeeper 경고는 "우클릭 → 열기" 안내 문서로 대응 (또는 추후 개발자 서명)
- 생성일은 `std::fs::Metadata::created()` 로 접근 가능

## 7. 엣지 케이스 정리

- 확장자가 없는 파일 → 이름 전체를 본문으로 취급
- `260231_xx` 같은 존재하지 않는 날짜 → 변환 불가 표시 후 건너뜀 (F-11)
- 파일명은 같고 확장자만 다른 파일 → 충돌 아님 (정상 처리)
- 순번 모드에서 생성일이 동일한 파일 → 파일명 사전순을 2차 정렬 기준으로 사용
- 읽기 전용 파일 / 권한 없는 폴더 → 실패 목록에 사유와 함께 표시
- 적용 도중 일부 실패 → 성공분은 유지, 실패분만 목록으로 안내 (Undo로 전체 복원 가능)

## 9. 성공 지표

- 파일 100개 리네이밍 작업 시간: 수작업 대비 90% 이상 단축

## 10. 오픈 퀘스천

1. `강남` → `강남역`처럼 **장소명 자체를 수정**하는 케이스도 있었는데, 이건 별도 "찾아 바꾸기" 기능이 필요한가? (v1 포함 여부 결정 필요)
2. 순번 모드에서 "최신 생성 순"이 최신 파일이 `01`인지, 오래된 파일이 `01`인지 기본값 확정 필요 (현재 기본: 최신 = 01)
3. 팀 배포 시 코드 서명/공증(notarization) 비용을 쓸지, 우클릭 열기 안내로 갈지
